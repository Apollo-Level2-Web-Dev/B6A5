# Project Overview

The **Event Planner & Participation System** is a secure, JWT‑protected web platform where registered users can:

*   Create, edit, and delete their own events as either **Public** (open to all) or **Private** (host approval required).
*   Optionally charge a **registration fee**; attendees must complete payment before joining.
*   Browse upcoming events via a homepage slider or detailed listings.
*   Join free public events instantly, or request access (and pay, if required) for paid or private events.
*   Manage participants: approve or reject requests, issue bans, or send direct invitations (with payment prompt when applicable).

An **Admin** oversees the entire ecosystem—monitoring events, users, and content to maintain a safe, well‑moderated community.

* * *

## Roles & Permissions

### Admin

*   Monitor all events and user activity
*   Delete inappropriate events or accounts

### User

*   Register and log in
*   **Event Management**: Create, update, and delete personal events (with optional registration fee)
*   **Event Discovery**: Browse and search Public and Private events
*   **Participation**:
    *   Join free Public events (auto‑accepted)
    *   Join Paid events (payment required → pending approval)
    *   Request access to Private events (payment pending if fee applies)
*   **Invitations**: Invite other users; invitees see a **Pay & Accept** button for paid events
*   **Participant Controls**: Approve or reject join requests; ban attendees

* * *

## Registration Fees & Payments

*   Creators specify a registration fee during event creation or editing.
*   Integrated payment flow (e.g., Stripe) collects fees before join requests.
*   All paid join attempts (Public or Private) generate **Pending** requests awaiting manual approval.

* * *

## Event Visibility & Participation Workflows

### 1\. Public Events (Free)

*   Visible to everyone
*   “Join” → instant acceptance

### 2\. Public Events (Paid)

*   Visible to everyone
*   “Join” → payment flow → **Pending** approval

### 3\. Private Events (Free)

*   Listed for logged‑in users
*   “Request to Join” → **Pending** approval

### 4\. Private Events (Paid)

*   Listed for logged‑in users
*   “Request to Join” → payment flow → **Pending** approval

* * *

## Member Invitations

*   Hosts can invite registered users to free or paid private events.
*   Invited users see a dashboard notification with **Pay & Accept** (for paid invites).
*   Upon payment, membership enters **Pending** state until host approval.

## Pages & Layout

1. **Global Navigation**: Home • Events • Login/Signup • Dashboard
2. **Footer**: About • Contact • Privacy Policy

### Home Page

*   Hero section featuring Admin‑selected event
*   Slider showcasing 9 upcoming public events (with fee badges)

### Events Page

*   Filters: Public Free • Public Paid • Private Free • Private Paid
*   Search bar (Title, Organizer)

### Event Details Page

*   Event metadata: Title, Date/Time, Venue/Link, Description, Organizer, Fee
*   Action button varies by type:
    *   Free Public → **Join**
    *   Paid Public → **Pay & Join**
    *   Free Private → **Request to Join**
    *   Paid Private → **Pay & Request**
*   Owner controls: Approve/Reject, Ban, Edit/Delete
*   Reviews & Ratings section (post‑event)

### Dashboard Page

*   **Sidebar**: My Events • Pending Invitations • My Reviews • Settings
*   **Main Area**:
    *   **My Events**: List, CRUD, payment statuses
    *   **Invitations**: Accept/Decline (with refund logic)
    *   **Reviews**: Edit/delete within review period
    *   **Settings**: Profile, Notifications
